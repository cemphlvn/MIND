MIND

Documentation

Overview

MIND is a local cognitive runtime. It accepts embeddings as experience and returns compressed hints. It does not generate language or reasoning.

Core loop

cr_state_update(state, embedding, dim, delta_t);
cr_state_query(state, query, dim, &hint);
cr_state_plasticity(state, &meta);

This loop is sufficient to integrate MIND into any system.

Plasticity

Plasticity represents how malleable the system is. High plasticity implies low commitment. Plasticity decays only through reinforcement.

Age

Age is continuous experiential time. It advances only via caller-supplied delta values. No wall-clock time is used.

Hints

Queries return hints, not answers. Each hint includes a confidence value derived from: similarity, reinforcement, and stability.

S2S Calibration

MIND exports calibration signals for server-to-server coevolution. Signals contain meta-cognitive state, not memory content. 

cr_calibration_t cal;
cr_state_calibration(state, &cal);
// cal.age, cal.plasticity, cal.velocity, cal.maturity

Determinism

Given identical inputs and ordering, MIND evolves identically across machines.

Persistence

State can be saved and restored explicitly. A saved state represents accumulated experience, not a model snapshot.
